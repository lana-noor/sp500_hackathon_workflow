S&P 500 Research Workflow — Architecture

505 S&P 500 companies  ·  11 GICS sectors  ·  January 2017 – December 2021  ·  5 sequential agents

🔍 User Query
  • Natural language research question about S&P 500 performance (2017–2021)
  • Examples: single stock deep dive, sector comparison, ranking, risk analysis, correlation study
  • Passed to portfolio_risk_workflow.py --query "..."
🤖 Agent 1  —  Query Planner FoundryChatClient

What It Does

  • Classifies query intent (sector_comparison, single_stock_deep_dive, ranking, risk_analysis, correlation_study, general)
  • Resolves tickers and sectors in scope
  • Generates 3–5 analytical questions (answerable from CSV data)
  • Generates 2–3 qualitative questions (for web research)
  • Sets the presentation angle for the final deck

Output JSON Keys

  • intent — query classification
  • scope.tickers / scope.sectors
  • time_window — start / end month
  • analytical_questions[]
  • qualitative_questions[]
  • presentation_angle
research plan JSON
tickerlist.csv  505 constituents
returns.csv  60 monthly returns
prices.csv  monthly snapshots
📊 Agent 2  —  Quantitative Analyst FoundryAgent + Code Interpreter ✓ Validated

What It Does

  • Loads all three CSV files in a Python sandbox (pandas / numpy)
  • Answers every analytical question with real computed values — no estimates
  • Computes: cumulative return, year-by-year annual returns, max drawdown, Sharpe ratio, annualised volatility, correlations, rankings
  • Output validated — rejects hallucinated templates, placeholder text, or missing findings key

Output JSON Keys

  • findings[] — question + answer + supporting_numbers
  • data_coverage — months, date range, ticker count
  • data_summary.top_performers[]
  • data_summary.bottom_performers[]
  • data_summary.key_stats
  • out_of_scope[] — unanswerable questions
findings JSON with real computed numbers
🔍 Agent 3  —  Search Query Expander FoundryChatClient

What It Does

  • Reads the research plan (Agent 1) and findings summary (Agent 2)
  • Generates 5–8 precise web search query strings for qualitative context
  • Queries describe business drivers, macro events, industry themes — not raw statistics
  • Each query targets a specific qualitative question with a preferred source type

Output JSON Keys

  • queries[].query_id
  • queries[].query_string — ready-to-pass search string
  • queries[].serves_question
  • queries[].preferred_source_type
  • grounding_note — how findings shaped queries
5–8 search query strings
Tavily Search API  (one call per query)
🌎 Agent 4  —  Web Researcher FoundryChatClient + Tavily ↻ loop per query

What It Does

  • Loops through every query from Agent 3
  • Calls Tavily Search API once per query (max 3 results, includes Tavily answer)
  • For each result set, calls FoundryChatClient to extract 2–4 specific factual claims
  • Skips paywalled, promotional, or off-topic sources
  • Records coverage_gap: true when no relevant sources found

Output JSON Keys

  • evidence_pack[].query_id
  • evidence_pack[].sources[].url
  • evidence_pack[].sources[].key_points[]
  • evidence_pack[].sources[].relevance
  • evidence_pack[].synthesis_note
  • coverage_gaps[] — topics with no web evidence
evidence pack with URLs and sourced claims
📚 Agent 5  —  Synthesizer FoundryChatClient

What It Does

  • Merges Agent 2 findings and Agent 4 evidence into a structured slide_deck_spec JSON
  • Designs 5–8 slides: title → exec summary → body slides → sources
  • Every factual claim backed by a number from Agent 2 or a URL from Agent 4
  • Year-by-year annual return data always placed in a dedicated comparison_table slide

Slide Types

  • title — deck title, subtitle, analysis period
  • exec_summary — 3–5 bullets with real numbers
  • comparison_table — metrics table with exact values
  • narrative — qualitative bullets with URL citations
  • sources — all unique URLs from evidence pack
slide_deck_spec JSON
⚙ generate_pptx() Deterministic Python function — not an LLM
  • Reads slide_deck_spec from Agent 5 — no model calls
  • Renders each slide type with python-pptx using the dark navy / gold design theme
  • Comparison tables rendered as styled pptx tables with alternating row fills
  • Footer on every body slide: dataset period, analysis date
